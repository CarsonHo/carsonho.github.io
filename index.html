    簡單來說, 就像TXT/WORD格式

    • TXT: 純文字
    • WORD: 要滑鼠左click右click
    • Markdown: 雙手不用離開鍵盤就能使用所有格式(如粗體等)

    好處：
    裝逼
    但個人更喜愛evernote

    什麼是HEXO?

    • 使用簡單
    • 方便快捷
    • 多主題 多功能

    如何設定Hexo博客？

    安裝程式

    • Github
    • Node.js (最新版)

    配置

    進入代碼庫創建頁面：
    在Repository name下填寫yourname.github.io
    要建立自己的 gitpage 也很簡單, 在你的 github 上新建一個項目, 項目名為: YOUR_GITHUB_USERNAME.github.io

    再到 Setting => Github Pages => Launch automatic page generator

    在cmder安裝Hexo

    npm install hexo --save

    查詢 Hexo是否安裝完成

    hexo -v

    初始化Hexo

    hexo init => npm install

    配置cmder

    首先，你需要為自己配置身份信息，然後輸入 

    git config --global user.name "yourname"  
git config --global user.email "youremail"

    然後輸入
    hexo clean=>hexo g=>hexo ｄ=>hexo s

    在瀏覽器打開 localhost:4000 即可打開網站

    更改主題

    如果你看中了那個主題, 只需要在 theme 目錄下 git clone 對應主題的倉庫, 然後把文件夾隨便改個名字, 然後在外面的 _config.yml 文件中的 theme: 後面加上那個文件夾的名字即可.

    網域名

    在GoDaddy買左一個網域 XXX.com 約$80/年

    • 到自己Repository github.com 的 accountName.github.io 新增一個檔案 CNAME, 內容是自己的網域名
    注意
    • CNAME 必須大草
    • A 主機IP(github): (@)
      192.30.252.153
      192.30.252.154

    • CNAME 設定 www主機 指向位置是 XXX.github.io
      約 10分鐘後即可看到成果

      Troubleshooting

    1. cannot get /about => hexo new page about
      輸入這句後 就能在 source folder建立about文件夾 的 index.md
    2. ERROR Deployer not found: git

    在 hexo3.0之後, 不僅 github 的 deploy 的 type 從 github 改為了 git, 其中 deploy 的功能也被單獨做成一個模塊, 需要另外安裝, 所以我們需要執行以下

    npm install hexo-deployer-git –save

    其中–save 參數是用於把模塊的版本號添加到 package.json 文件中的依賴裡( dependences ), 否則在安裝完之後需要手動添加.
